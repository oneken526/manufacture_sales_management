# 承認フロー 共通仕様書

作成日: 2026-05-17  
対象システム: 製造業向け販売管理システム  
関連要件: REQ-015, REQ-017, REQ-021, REQ-022, REQ-025, REQ-026, EDGE-001

---

## 1. この仕様の対象

このシステムでは、**見積書**と**受注**の2種類のドキュメントに同じ承認フローが適用されます。  
本仕様書はその共通ルールをまとめたものです。

| ドキュメント | 適用要件 |
|---|---|
| 見積書 | REQ-015 |
| 受注 | REQ-017 |

---

## 2. ステータスの種類

| ステータス | 表示名 | 意味 |
|---|---|---|
| `draft` | 下書き | 作成直後の編集可能な状態 |
| `pending` | 承認待ち | 承認申請済みで管理者の判断を待っている状態 |
| `approved` | 承認済み | 管理者が承認した正式な状態 |
| `rejected` | 却下 | 管理者が却下した状態。修正して再申請が必要 |

---

## 3. ステータス遷移図

```
[作成]
  ↓
draft（下書き）
  ↓ 承認申請（salesが操作）
pending（承認待ち）
  ↓                  ↓
approved（承認済み）  rejected（却下）
                       ↓
                     draft（下書き）に戻して再申請可能
```

> **却下後の扱い**  
> 却下されたドキュメントは `rejected` のまま残ります。  
> 申請者が内容を修正して再申請したい場合は、複製して新しい下書きを作成してください。

---

## 4. 各ステータスでできる操作

### 4-1. draft（下書き）

| 操作 | sales | admin |
|---|---|---|
| 内容を編集する | ○ | ○ |
| 承認申請する | ○ | ○ |
| 削除する | ○ | ○ |

### 4-2. pending（承認待ち）

| 操作 | sales | admin |
|---|---|---|
| 内容を編集する | ✗ | ✗ |
| 承認する | ✗ | ○ |
| 却下する | ✗ | ○ |

> **編集不可の理由（REQ-026）**  
> 承認待ち中は申請者本人であっても編集できません。  
> 内容を変更したい場合は、admin に却下を依頼してください。

### 4-3. approved（承認済み）

| 操作 | sales | admin |
|---|---|---|
| 内容を編集する | ✗ | ✗ |
| 受注に変換する（見積書のみ） | ○ | ○ |
| PDFをダウンロードする | ○ | ○ |

### 4-4. rejected（却下）

| 操作 | sales | admin |
|---|---|---|
| 内容を編集する | ✗ | ✗ |
| 複製して新しい下書きを作る | ○ | ○ |
| 削除する | ○ | ○ |

---

## 5. 承認ルール

- 承認者は **admin ロールを持つユーザー** のみです（REQ-025）。
- 承認は **一段階承認**（複数人による多段階承認は行いません）（REQ-025）。
- 誰でも承認できます（特定の担当者への割り当て機能はありません）。

---

## 6. メール通知

承認フローの操作に応じて、以下のタイミングでメールが送信されます（REQ-021, REQ-022）。

| タイミング | 送信先 | 内容 |
|---|---|---|
| 承認申請したとき | admin ロールの全ユーザー | 承認申請が届いた旨と対象ドキュメントのリンク |
| 承認されたとき | 申請者（sales） | 承認完了の通知 |
| 却下されたとき | 申請者（sales） | 却下の通知と却下理由 |

> **申請失敗時の動作（EDGE-001）**  
> メール送信が失敗した場合は、承認申請の処理ごとロールバックされます。  
> ステータスは `draft` のままとなり、エラーメッセージが表示されます。

---

## 7. 却下コメント

- admin が却下する際、**却下理由（テキスト）を入力できます**（任意）。
- 入力した却下理由は申請者の画面に表示され、メール通知にも含まれます。
- 却下理由はデータベースの `rejection_reason` カラムに保存されます。
