# 得意先管理 機能仕様書

作成日: 2026-05-13  
対象システム: 製造業向け販売管理システム  
関連要件: REQ-005, REQ-006, REQ-020, REQ-024, EDGE-007, EDGE-008

---

## 1. この機能の目的

得意先（販売先のお客様）の基本情報を登録・管理する機能です。  
ここで登録した得意先情報は、見積・受注・請求などすべての販売業務の起点となります。

---

## 2. 利用できるユーザー

| ロール | 操作 |
|---|---|
| 管理者（admin） | すべての操作が可能 |
| 営業（sales） | すべての操作が可能 |
| 製造（manufacture） | 利用不可 |
| 倉庫（warehouse） | 利用不可 |

---

## 3. 画面一覧

| 画面名 | 説明 |
|---|---|
| 得意先一覧 | 登録されている得意先を一覧で確認する |
| 得意先登録 | 新しい得意先を登録する |
| 得意先編集 | 登録済みの得意先情報を修正する |
| 得意先詳細 | 得意先の情報と特別単価の一覧を確認する |

---

## 4. 得意先の登録項目

### 4-1. 基本情報

| 項目名 | 必須 | 入力内容 | 補足・制約 |
|---|---|---|---|
| 社名 | ○ | 得意先の会社名 | 最大255文字 |
| フリガナ | - | 社名のフリガナ | 最大255文字。一覧での検索に使用 |
| 郵便番号 | - | 例: 123-4567 | ハイフンあり・なし両方可 |
| 住所 | - | 都道府県から番地まで | 最大500文字 |
| 電話番号 | - | 例: 03-1234-5678 | |
| メールアドレス | - | 例: info@example.com | メール形式で入力 |
| 締日 | ○ | 月次請求の締め日 | 1〜28 の整数、または 99（月末）を入力。29・30・31 は入力不可 |
| 与信限度額 | ○ | 受注可能な上限金額（円） | 0 を入力した場合は「制限なし」として扱う |
| 備考 | - | 特記事項など自由記入 | |

> **締日について**  
> 月によって日数が異なるため、29・30・31 は設定できません。  
> 月末締めにしたい場合は「99」を入力してください。

> **与信限度額について**  
> 受注登録時にこの金額を超えると警告が表示されます。ただし警告のみで受注を止めることはしません。  
> 0 を設定すると与信チェックはスキップされます。

---

## 5. 機能詳細

### 5-1. 一覧表示

- 登録済みの得意先を一覧で表示します。
- 1ページあたり **20件** 表示し、ページ切替で残りを確認できます。
- **社名** または **フリガナ** で絞り込み検索できます。
- 削除済みの得意先は一覧に表示されません（論理削除）。

### 5-2. 新規登録

- 「新規登録」ボタンから登録フォームを開きます。
- 必須項目（社名・締日・与信限度額）が未入力の場合はエラーを表示します。
- 登録が成功すると一覧画面に戻り、完了メッセージが表示されます。

### 5-3. 編集

- 一覧または詳細画面の「編集」ボタンから編集フォームを開きます。
- 修正後に「保存」すると更新が反映されます。

### 5-4. 削除

- 削除しても **データは完全には消えません**（論理削除）。  
  画面上は非表示になりますが、過去の見積・受注・請求データとの紐付きは保持されます。

## 6. 得意先別 特別単価

### 6-1. 概要

得意先ごとに、特定の商品を通常単価より安く（または高く）設定できる機能です。  
特別単価が設定された商品は、見積・受注の明細入力時に自動で適用されます。

### 6-2. 登録項目

| 項目名 | 必須 | 内容 |
|---|---|---|
| 商品 | ○ | 特別単価を設定する商品 |
| 特別単価 | ○ | 適用する単価（円）。0以上の数値 |

### 6-3. 制約

- 同じ得意先に対して **同じ商品の特別単価は1件のみ** 登録できます。  
  重複して登録しようとするとエラーになります。
- 特別単価を削除すると、次回以降の見積・受注では商品マスタの **標準単価** が適用されます。

### 6-4. 見積・受注への自動反映

得意先を選択した際、商品ごとに以下の優先順位で単価が自動セットされます。

```
1. 得意先別 特別単価（登録されている場合）
2. 商品マスタの 標準単価（特別単価がない場合）
```

---

## 7. 他機能との連携

| 機能 | 連携内容 |
|---|---|
| 見積管理 | 得意先選択時に特別単価を自動適用 |
| 受注管理 | 得意先選択時に特別単価を自動適用。与信限度額超過時に警告表示 |
| 請求管理 | 得意先の締日をもとに月次請求書を生成 |

---

## 8. エラーと注意事項

| 状況 | システムの動作 |
|---|---|
| 社名が未入力 | 登録・更新を受け付けず、エラーメッセージを表示 |
| 締日に29〜31を入力 | バリデーションエラー。1〜28または99を入力するよう案内 |
| 与信限度額にマイナス値を入力 | バリデーションエラー。0以上の値を入力するよう案内 |
| 同一商品の特別単価を重複登録 | バリデーションエラー。既存の特別単価の編集を案内 |
| 受注時に与信限度額を超過 | 警告を表示するが受注は続行可能（受注を止めない） |
| 与信限度額が0の得意先 | 与信チェックをスキップ（制限なしとみなす） |

---

## 9. 用語説明

| 用語 | 意味 |
|---|---|
| 締日 | 月次請求の集計を締める日。例: 締日=20 なら毎月20日締め |
| 与信限度額 | その得意先に対して受注できる上限金額の目安 |
| 特別単価 | 得意先ごとに設定する商品の個別単価。標準単価より優先される |
| 論理削除 | 画面上は削除されたように見えるが、データ自体はシステム内に保持される削除方式 |
| ソフトデリート | 論理削除と同じ意味（システム用語） |
