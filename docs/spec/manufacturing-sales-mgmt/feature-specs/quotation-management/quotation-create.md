# 見積書登録画面 仕様書

作成日: 2026-05-17  
対象システム: 製造業向け販売管理システム  
関連要件: REQ-013, REQ-014, REQ-020, REQ-021, REQ-025

---

## 1. この画面の目的

新しい見積書を作成する画面です。  
得意先・有効期限・明細（商品・数量・単価）を入力して保存すると、**`draft`（下書き）** 状態の見積書が作成されます。

---

## 2. アクセスできるユーザー

| ロール | アクセス |
|---|---|
| 管理者（admin） | ○ |
| 営業（sales） | ○ |
| 製造（manufacture） | ✗ |
| 倉庫（warehouse） | ✗ |

---

## 3. 画面構成

```
[見積書登録]
┌─────────────────────────────────┐
│ 基本情報                         │
│  見積番号: Q202605-001（自動）    │
│  得意先:   [セレクトボックス]     │
│  有効期限: [日付入力]            │
│  作成者:   山田 太郎（自動）      │
├─────────────────────────────────┤
│ 明細                             │
│  [商品]    [数量]  [単価]  [金額] │
│  ○○○      10     1,000   10,000 │
│  ○○○       5     2,000   10,000 │
│                   [行を追加]     │
├─────────────────────────────────┤
│ 合計金額: 20,000 円               │
├─────────────────────────────────┤
│          [キャンセル] [保存する]  │
└─────────────────────────────────┘
```

---

## 4. 基本情報の入力項目

| 項目名 | 種別 | 必須 | 入力内容 | 補足・制約 |
|---|---|---|---|---|
| 見積番号 | テキスト（表示のみ） | 自動 | 例: Q202605-001 | システムが自動採番。手入力不可 |
| 得意先 | セレクトボックス | ○ | 登録済みの得意先から選択 | 選択後、明細の単価に特別単価を自動反映（Ajax） |
| 有効期限 | 日付入力 | - | 見積書の有効期限 | 任意。期限を過ぎても操作は制限されない |
| 作成者 | テキスト（表示のみ） | 自動 | ログイン中のユーザー名 | 自動セット。変更不可 |

### 見積番号の採番形式

`Q{YYYYMM}-{3桁連番}` の形式で自動採番されます。

- 例: 2026年5月に最初に作成した見積書 → `Q202605-001`、次 → `Q202605-002`
- 翌月になると連番は `001` からリセットされます。
- 採番は保存成功時に確定します（フォーム表示時点では仮表示でも可）。

---

## 5. 明細の入力項目

1件の見積書に複数の明細行を登録できます。  
明細が **0行の場合は保存できません**。

| 項目名 | 種別 | 必須 | 入力内容 | 補足・制約 |
|---|---|---|---|---|
| 商品 | セレクトボックス | ○ | 登録済みの商品から選択 | 選択後、単価に標準単価を自動セット |
| 数量 | 数値入力 | ○ | 数量（整数または小数） | 1 以上の値を入力すること（EDGE-006） |
| 単価 | 数値入力 | ○ | 販売単価（円） | 得意先選択時に自動セット（後述）。手動変更も可能 |
| 金額 | テキスト（表示のみ） | 自動 | 数量 × 単価 | 数量・単価の入力に連動して自動計算。入力不可 |
| 削除 | ボタン | - | 当該行を削除 | 1行のみの場合はボタンを無効化または非表示にする |

---

## 6. 単価の自動反映（REQ-020）

得意先を選択すると、Ajax で単価が自動セットされます。

| 条件 | セットされる単価 |
|---|---|
| その得意先に**特別単価**が登録されている | 特別単価 |
| 特別単価が登録されていない | 商品マスタの**標準単価** |

- 得意先を変更すると、明細に入力済みの単価はすべて再取得して上書きされます。
- 自動セット後に手動で単価を変更することもできます。
- Ajax 通信中はローディング表示を出すなど、操作中であることをユーザーに伝えること。

---

## 7. 合計金額

画面下部に全明細の金額合計を表示します。

- 計算式: `SUM(各明細の 数量 × 単価)`
- 数量・単価の変更に連動してリアルタイムで更新されます。
- 表示単位は円（税抜）。

---

## 8. 明細行の操作

| 操作 | 動作 |
|---|---|
| 「行を追加」ボタン | 明細に空行を1行追加する |
| 「削除」ボタン（行ごと） | 当該行を削除する。1行のみの場合は無効化 |

---

## 9. 保存・キャンセル

| ボタン | 動作 |
|---|---|
| 保存する | バリデーションを実行し、問題なければ見積書を保存する。保存成功後は**見積書詳細画面**へリダイレクト |
| キャンセル | 変更を破棄して**見積書一覧画面**へ戻る |

- 保存後の見積書ステータスは `draft`（下書き）になります。
- 見積番号は保存成功時に確定・採番されます。

---

## 10. バリデーション

### 保存ボタン押下時に検証する内容

| 項目 | ルール | エラーメッセージ（例） |
|---|---|---|
| 得意先 | 必須。customers テーブルに存在すること | 「得意先を選択してください」 |
| 有効期限 | 任意。入力する場合は日付形式（Y-m-d） | 「有効期限を正しい日付形式で入力してください」 |
| 明細行 | 1行以上存在すること | 「明細を1行以上入力してください」 |
| 商品 | 各明細で必須。products テーブルに存在すること | 「商品を選択してください」 |
| 数量 | 各明細で必須。1以上の数値 | 「数量は1以上の値を入力してください」 |
| 単価 | 各明細で必須。0以上の数値 | 「単価を入力してください」 |

- エラーがある場合は入力内容を保持した状態でフォームを再表示します。
- エラーメッセージは該当項目の近くに表示します。

---

## 11. 画面遷移

```
見積書一覧画面
  └─ [新規登録] ─→ 見積書登録画面（この画面）
                       │
                       ├─ [保存する] ─→ 見積書詳細画面（作成した見積書）
                       └─ [キャンセル] ─→ 見積書一覧画面
```

---

## 12. 注意事項

- 得意先を**変更した場合**、明細の単価はすべて再取得されます。意図せず単価が変わる可能性があるため、変更時に確認ダイアログを表示することを推奨します。
- 商品セレクトボックスは、廃番・無効化された商品を除いた一覧を表示します。
- 数量・単価の入力欄は数値のみ受け付けます。

---

## 13. 用語説明

| 用語 | 意味 |
|---|---|
| 見積番号 | 見積書を一意に識別する番号。形式は `Q{YYYYMM}-{3桁連番}`（例: Q202605-001） |
| 有効期限 | 見積書の有効期間の終了日。期限を過ぎても操作は制限されない |
| 特別単価 | 得意先ごとに設定する商品の個別単価。標準単価より優先される |
| 標準単価 | 商品マスタに設定されたデフォルトの販売単価 |
| 合計金額 | 明細の「数量 × 単価」を合計した金額（税抜） |
| draft | 作成直後の編集可能な状態。承認申請するまでこのステータスで維持される |
