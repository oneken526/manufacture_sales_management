# 見積書一覧画面 仕様書

作成日: 2026-05-17  
対象システム: 製造業向け販売管理システム  
関連要件: REQ-013, REQ-015

---

## 1. この画面の目的

登録されている見積書をまとめて確認する画面です。  
検索・絞り込みで目的の見積書を探し、詳細確認・編集・PDF出力などの操作を行う起点となります。

---

## 2. アクセスできるユーザー

| ロール | アクセス |
|---|---|
| 管理者（admin） | ○ |
| 営業（sales） | ○ |
| 製造（manufacture） | ✗ |
| 倉庫（warehouse） | ✗ |

---

## 3. 一覧の表示項目

| 列名 | 内容 | 備考 |
|---|---|---|
| 見積番号 | 例: Q202605-001 | クリックで詳細画面に遷移 |
| 得意先名 | | |
| 合計金額 | 明細の金額合計（税抜・円） | |
| 有効期限 | 見積書の有効期限（日付） | 期限切れは視覚的に強調表示 |
| ステータス | 下書き／承認待ち／承認済み／却下 | バッジ形式で表示（後述） |
| 作成者 | 見積書を作成したユーザー名 | |
| 作成日 | | |
| 操作 | 詳細・編集・複製ボタン | ステータスにより表示が変わる（後述） |

- 1ページあたり **20件** 表示します。
- 削除済みの見積書は表示されません（論理削除）。
- デフォルトは**作成日の降順**（新しい順）で表示します。

---

## 4. 検索・絞り込み

画面上部の検索フォームから以下の条件で絞り込みできます。  
複数条件は AND で適用されます。

| 検索条件 | 入力方式 | 一致方式 |
|---|---|---|
| 見積番号 | テキスト入力 | 部分一致 |
| 得意先名 | テキスト入力 | 部分一致 |
| ステータス | セレクトボックス | 完全一致 |
| 作成日（開始） | 日付入力 | 以降 |
| 作成日（終了） | 日付入力 | 以前 |

- 「検索」ボタンで絞り込みを実行します。
- 「クリア」ボタンで条件をリセットして全件表示に戻ります。

---

## 5. ステータスバッジ

ステータスはバッジ形式で表示されます。

| ステータス | 表示名 | バッジの色 |
|---|---|---|
| `draft` | 下書き | グレー |
| `pending` | 承認待ち | 黄色 |
| `approved` | 承認済み | 緑 |
| `rejected` | 却下 | 赤 |

> ステータスの意味・遷移ルール・承認操作の詳細は  
> [承認フロー 共通仕様書](../approval-flow.md) を参照してください。

---

## 6. 操作ボタン

各行の「操作」列に表示されるボタンは、ステータスによって変わります。

| ボタン | 表示条件 | 遷移先・動作 |
|---|---|---|
| 詳細 | 常に表示 | 見積書詳細画面へ遷移 |
| 編集 | `draft` のみ | 見積書編集画面へ遷移 |
| 複製 | 常に表示 | 内容をコピーした新しい下書きを作成 |

画面右上には以下のボタンがあります。

| ボタン | 動作 |
|---|---|
| 新規登録 | 見積書登録画面へ遷移 |

---

## 7. 注意事項

- `pending`（承認待ち）の見積書は編集ボタンが表示されません。内容を変更したい場合は、admin に却下を依頼してください。
- 有効期限を過ぎた見積書も引き続き一覧に表示されます。期限切れであっても操作は制限されません。

---

## 8. 用語説明

| 用語 | 意味 |
|---|---|
| 見積番号 | 見積書を一意に識別する番号。形式は `Q{YYYYMM}-{3桁連番}`（例: Q202605-001） |
| 有効期限 | 見積書の有効期間の終了日 |
| 合計金額 | 明細の「数量 × 単価」を合計した金額（税抜） |
| 論理削除 | 画面上は表示されないが、データ自体はシステム内に保持される削除方式 |
