## 開発ルール

- **CLAUDE.md 準拠**: 実装作業を行ったら必ず `.claude/PROGRESS.md` の該当項目を更新すること（⬜ → 🚧 → ✅ およびサマリ数値）。
- ロール: `admin` / `sales` / `manufacture` / `warehouse`
- 信頼性レベルの凡例:
  - 🔵 PRD・ヒアリングで確認済みの確実な要件
  - 🟡 PRD・設計文書から妥当な推測
  - 🔴 推測による要件（PRD・ヒアリングにない）

## ヒアリング確定事項サマリー

| # | 質問項目 | 確定内容 |
|---|---|---|
| Q1 | 承認ワークフロー | 一段階承認（管理者一人が承認/却下） |
| Q2 | PDF言語 | 日本語のみ |
| Q3 | 製造指示の粒度 | 受注単位（1受注 = 1製造指示） |
| Q4 | 在庫引き当てルール | 手動引き当て（担当者がロットを手動選択） |
| Q5 | 月次請求書生成タイミング | 手動実行（管理者が画面からボタン操作） |
| Q6 | 入金消込（分割充当） | 必要（1入金を複数請求書に按分） |
| Q7 | 通知機能 | メール通知あり（承認申請・承認完了時） |
| Q8 | 与信限度チェック | 警告表示のみ（受注は止めない） |
| Q9 | 売上集計レポート | MVPスコープに含める（月次・得意先別・商品別） |
| Q10 | Must Have確認 | 全機能MVP（認証・見積・受注・製造・出荷・在庫・請求・入金） |
