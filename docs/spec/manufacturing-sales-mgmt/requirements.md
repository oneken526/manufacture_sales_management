# 製造業向け販売管理システム 要件定義書

作成日: 2026-03-29

---

## 概要

製造業向けの販売管理システム。見積・受注・製造指示・出荷・在庫・請求・入金の一連の業務フローを管理する。
Laravel 13.0 / PHP 8.3+ / MySQL 8.0+ を基盤とし、Spatie Permission によるロールベースアクセス制御を採用する。

**信頼性レベル凡例:**
- 🔵 PRD・ヒアリングで確認済みの確実な要件
- 🟡 PRD・設計文書から妥当な推測
- 🔴 推測による要件（PRD・ヒアリングにない）

### Phase 2: 見積・受注管理

#### 通常要件

| ID | 要件 | 信頼性 | 出典 |
|---|---|---|---|
| REQ-013 | システムは見積書の作成・編集・複製機能を提供しなければならない。 | 🔵 | PRD Phase 2 |
| REQ-014 | システムは見積書に明細行（商品・数量・単価・金額）を複数登録できなければならない。 | 🔵 | PRD Phase 2 |
| REQ-015 | システムは見積書を `draft` → `pending` → `approved` または `rejected` のステータスで管理しなければならない。 | 🔵 | PRD Phase 2 / ヒアリングQ1 |
| REQ-016 | システムは受注の作成・編集機能を提供しなければならない。 | 🔵 | PRD Phase 2 |
| REQ-017 | システムは受注を `draft` → `pending` → `approved` または `rejected` のステータスで管理しなければならない。 | 🔵 | PRD Phase 2 / ヒアリングQ1 |
| REQ-018 | システムは承認済み見積書のみを受注に変換できなければならない。 | 🔵 | PRD Phase 2 |
| REQ-019 | システムは受注残（受注数量 - 出荷数量）を管理しなければならない。 | 🔵 | PRD Phase 2 |

#### 条件付き要件

| ID | 要件 | 信頼性 | 出典 |
|---|---|---|---|
| REQ-020 | 得意先を選択したとき、システムはその得意先の特別単価を見積明細の単価に自動反映しなければならない。 | 🔵 | PRD Phase 2 |
| REQ-021 | 承認申請が行われたとき、システムは対象の管理者にメール通知を送信しなければならない。 | 🔵 | PRD Phase 2 / ヒアリングQ7 |
| REQ-022 | 承認または却下が完了したとき、システムは申請者にメール通知を送信しなければならない。 | 🔵 | PRD Phase 2 / ヒアリングQ7 |
| REQ-023 | 受注が承認されたとき、システムは自動的に対応する製造指示を1件発行しなければならない。 | 🔵 | PRD Phase 2 / ヒアリングQ3 |
| REQ-024 | 受注登録または編集時に得意先の与信限度額を超過したとき、システムは警告を表示しなければならない（受注は止めない）。 | 🔵 | PRD Phase 2 / ヒアリングQ8 |

#### 制約要件

| ID | 要件 | 信頼性 | 出典 |
|---|---|---|---|
| REQ-025 | 承認ワークフローは一段階承認（admin ロールを持つ一人の管理者による承認）でなければならない。 | 🔵 | ヒアリングQ1 |
| REQ-026 | `pending` 状態の見積書・受注は申請者本人であっても編集できないものとする。 | 🔵 | ヒアリング確認済み |

| ID | 要件 | 信頼性 | 出典 |

## Edgeケース

### エラー処理

| ID | ケース | 期待動作 | 信頼性 | 出典 |
|---|---|---|---|---|
| EDGE-001 | 承認申請メール送信失敗 | メール送信失敗した場合は承認申請処理ごとロールバックし、エラーメッセージを表示する。 | 🔵 | ヒアリング確認済み |
| EDGE-002 | 在庫数量が0のロットへの出荷引き当て | エラーメッセージを表示し、引き当てを拒否する。 | 🔵 | PRD Phase 3 |
| EDGE-003 | PDF出力中にデータが変更された場合 | PDF生成開始時点のスナップショットデータを使用してPDFを生成する。 | 🔵 | ヒアリング確認済み |
| EDGE-004 | 月次請求書の重複生成 | 同一得意先・同一締め月に対して2件目以降の生成は警告を表示し、確認後のみ上書き可能とする。 | 🔵 | ヒアリング確認済み |
| EDGE-005 | 入金額が請求書合計を超過した場合の消込 | 超過分は前払金または未充当として管理し、次回請求書に充当可能とする。 | 🔵 | ヒアリング確認済み |
