## エピック1: 認証・ロール管理

### US-001 ログイン
**As a** システムユーザー（全ロール）,
**I want to** メールアドレスとパスワードでログインしたい,
**So that** 自分のロールに応じたシステム機能を利用できる。

- 受け入れ基準:
  - 正しい認証情報でログインするとダッシュボードに遷移する
  - 誤った認証情報ではエラーメッセージが表示される
  - ログアウト操作でセッションが破棄されログイン画面に戻る
- 信頼性: 🔵 | 出典: PRD Phase 1

---

### US-002 パスワードリセット
**As a** パスワードを忘れたユーザー,
**I want to** メールリンクからパスワードをリセットしたい,
**So that** 自分でパスワードを再設定してログインできる。

- 受け入れ基準:
  - 登録済みメールアドレスを入力するとリセットリンクが送信される
  - リンクから新パスワードを設定できる
- 信頼性: 🔵 | 出典: PRD Phase 1

---

### US-003 ロール別ダッシュボード
**As a** ログイン済みユーザー,
**I want to** 自分のロールに合ったダッシュボードを見たい,
**So that** 自分の業務に関係する情報だけを素早く確認できる。

- 受け入れ基準:
  - admin: 全体サマリー（受注件数・承認待ち件数・売掛残高など）
  - sales: 担当見積・受注一覧
  - manufacture: 製造指示一覧・進捗状況
  - warehouse: 出荷待ち一覧・在庫アラート
- 信頼性: 🔵 | 出典: PRD Phase 1

---

### US-004 ロール管理（admin）
**As a** admin,
**I want to** ユーザーにロールを付与・変更したい,
**So that** 各ユーザーが適切な権限でシステムを利用できる。

- 受け入れ基準:
  - admin / sales / manufacture / warehouse の4ロールを付与・変更できる
  - ロールなしのユーザーは保護ルートにアクセスできない
- 信頼性: 🔵 | 出典: PRD Phase 1 / ヒアリングQ10

---

## エピック2: マスタ管理

### US-005 得意先マスタ管理
**As a** admin または sales,
**I want to** 得意先情報を登録・編集・削除したい,
**So that** 見積・受注・請求で正しい得意先情報を使用できる。

- 受け入れ基準:
  - 社名・住所・締日・与信限度額・連絡先メールを管理できる
  - 削除はソフトデリート（データは論理削除）
- 信頼性: 🔵 | 出典: PRD Phase 1

---

### US-006 得意先別特別単価管理
**As a** admin または sales,
**I want to** 得意先ごとに商品の特別単価を設定したい,
**So that** 見積作成時に自動的に正しい単価が反映される。

- 受け入れ基準:
  - 得意先×商品の組み合わせで特別単価を登録できる
  - 特別単価が存在する場合、見積明細の単価に自動反映される
- 信頼性: 🔵 | 出典: PRD Phase 1

---

### US-007 商品マスタ管理
**As a** admin,

**So that** 見積・受注・在庫管理で一貫した商品情報を使用できる。

- 受け入れ基準:
  - 商品カテゴリを作成・編集・削除できる
  - 商品を作成・編集・削除（ソフトデリート）できる
  - 商品に対してカテゴリを紐付けられる
- 信頼性: 🔵 | 出典: PRD Phase 1

---

### US-008 倉庫マスタ管理
**As a** admin,
**I want to** 倉庫情報を管理したい,
**So that** 在庫管理・出荷登録で正しい倉庫を指定できる。

- 受け入れ基準:
  - 倉庫名・ロケーション情報を登録・編集できる
- 信頼性: 🔵 | 出典: PRD Phase 1

---

### US-009 ユーザーマスタ管理
**As a** admin,
**I want to** ユーザーアカウントを管理したい,
**So that** 従業員が適切な権限でシステムを利用できる。

- 受け入れ基準:
  - 氏名・メールアドレス・ロールを設定してユーザーを作成できる
  - ユーザーを無効化できる（ソフトデリートまたはフラグ管理）
- 信頼性: 🔵 | 出典: PRD Phase 1

## エピック3: 見積管理

### US-010 見積書作成
**As a** sales,
**I want to** 得意先を選択して見積書を作成したい,
**So that** 顧客に正式な見積書を提出できる。

- 受け入れ基準:
  - 得意先・有効期限・明細（商品・数量・単価）を入力できる
  - 得意先選択時に特別単価が自動反映される
  - 保存すると `draft` ステータスで作成される
- 信頼性: 🔵 | 出典: PRD Phase 2

---

### US-011 見積書編集・複製
**As a** sales,
**I want to** 既存の見積書を編集・複製したい,
**So that** 修正や類似見積の作成を効率的に行える。

- 受け入れ基準:
  - `draft` 状態の見積書は編集できる
  - `pending` 状態の見積書は編集できない（警告表示）
  - 複製操作で新しい `draft` 状態の見積書が作成される
- 信頼性: 🔵 | 出典: PRD Phase 2

---

### US-012 見積書承認申請
**As a** sales,
**I want to** 作成した見積書を承認申請したい,
**So that** 管理者に承認してもらい、正式な見積書として使用できる。

- 受け入れ基準:
  - 申請操作でステータスが `draft` → `pending` に変わる
  - 申請時に admin ロールのユーザーにメール通知が送信される
- 信頼性: 🔵 | 出典: PRD Phase 2 / ヒアリングQ7

---

### US-013 見積書承認・却下（admin）
**As a** admin,
**I want to** 申請された見積書を承認または却下したい,
**So that** 見積書の内容を確認し品質を担保できる。

- 受け入れ基準:
  - `pending` 状態の見積書を承認（`approved`）または却下（`rejected`）できる
  - 承認・却下時に申請者にメール通知が送信される
  - 却下時にコメント（理由）を入力できる
- 信頼性: 🔵 | 出典: PRD Phase 2 / ヒアリングQ1・Q7

---

### US-014 見積書PDF出力
**As a** sales,
**I want to** 見積書をPDFでダウンロードしたい,
**So that** 顧客にPDF形式で見積書を提出できる。

- 受け入れ基準:
  - 見積書詳細ページからPDFをダウンロードできる
  - PDFは日本語で出力される
- 信頼性: 🔵 | 出典: PRD Phase 5 / ヒアリングQ2

---

### US-015 見積書から受注への変換
**As a** sales,
**I want to** 承認済み見積書から受注を作成したい,
**So that** 顧客から受注を受けた際にスムーズに受注登録できる。

- 受け入れ基準:
  - `approved` 状態の見積書のみ受注変換ボタンが表示される
  - 変換すると見積内容が受注にコピーされ `draft` 状態で作成される
- 信頼性: 🔵 | 出典: PRD Phase 2

---

## エピック4: 受注管理

### US-016 受注作成・編集
**As a** sales,
**I want to** 受注を作成・編集したい,
**So that** 顧客からの注文を正確に記録できる。

- 受け入れ基準:
  - 得意先・納期・明細（商品・数量・単価）を入力できる
  - `draft` 状態の受注は編集できる
- 信頼性: 🔵 | 出典: PRD Phase 2

---

### US-017 与信限度チェック
**As a** sales,
**I want to** 受注登録時に与信限度超過の警告を見たい,
**So that** 与信リスクを認識した上で受注判断ができる。

- 受け入れ基準:
  - 受注金額が与信限度額を超過した場合に画面上に警告メッセージが表示される
  - 警告が表示されても受注登録は続行できる（強制停止しない）
- 信頼性: 🔵 | 出典: PRD Phase 2 / ヒアリングQ8

---

### US-018 受注承認申請・承認
**As a** sales / admin,
**I want to** 受注を承認フローで処理したい,
**So that** 受注内容を確認・承認して製造指示を発行できる。

- 受け入れ基準:
  - sales が承認申請すると `draft` → `pending` に変わる
  - 申請時に admin にメール通知が送信される
  - admin が承認すると `pending` → `approved` に変わる
  - 承認時に申請者にメール通知が送信される
  - admin が却下すると `pending` → `rejected` に変わる
- 信頼性: 🔵 | 出典: PRD Phase 2 / ヒアリングQ1・Q7

---

### US-019 受注承認時の製造指示自動発行
**As a** admin,
**I want to** 受注を承認したときに製造指示が自動発行されるようにしたい,
**So that** 承認と同時に製造部門が作業を開始できる。

- 受け入れ基準:
  - 受注承認と同時に製造指示が1件自動生成される
  - 製造指示には製番（製造番号）が自動採番される
  - 製造指示の初期ステータスは `pending`
- 信頼性: 🔵 | 出典: PRD Phase 2 / ヒアリングQ3

---

### US-020 受注残管理
**As a** sales または admin,
**I want to** 受注の残数量（未出荷数量）を確認したい,
**So that** 出荷状況と未出荷分を把握できる。

- 受け入れ基準:
  - 受注一覧・詳細に受注残（受注数量 - 出荷数量）が表示される
  - 出荷登録のたびに受注残がリアルタイムに更新される
- 信頼性: 🔵 | 出典: PRD Phase 2

---

## エピック5: 製造指示・進捗管理

### US-021 製造指示一覧・詳細確認
**As a** manufacture,
**I want to** 製造指示の一覧と詳細を確認したい,
**So that** 自分が担当する製造作業の内容と優先度を把握できる。

- 受け入れ基準:
  - 製造指示一覧にステータス・製番・受注情報・納期が表示される
  - 詳細ページで製品名・数量・関連受注情報を確認できる
- 信頼性: 🔵 | 出典: PRD Phase 3

---

### US-022 製造進捗ステータス更新
**As a** manufacture,
**I want to** 製造指示のステータスを更新したい,
**So that** 製造の進捗状況を正確に記録・共有できる。

- 受け入れ基準:
  - `pending` → `in_progress` → `completed` の順でステータスを更新できる
  - `cancelled` への変更は admin のみが実行できる
  - ステータス変更時に変更日時が記録される
- 信頼性: 🔵 | 出典: PRD Phase 3

---

### US-023 製造完了時の在庫入庫連動
**As a** manufacture,
**I want to** 製造完了時に在庫が自動的に入庫されるようにしたい,
**So that** 手動で在庫登録する手間なく在庫数量が更新される。

- 受け入れ基準:
  - 製造指示を `completed` にすると指定ロットで在庫が自動入庫される
  - 入庫時にロット番号・製造日・有効期限・数量を入力できる
  - 入庫先の倉庫を選択できる
- 信頼性: 🔵 | 出典: PRD Phase 3

---

### US-024 製番トレーサビリティ
**As a** manufacture または admin,
**I want to** 製番から製造指示・受注・出荷・在庫を追跡したい,
**So that** 製品のトレーサビリティを確保できる。

- 受け入れ基準:
  - 製番から関連する受注・在庫ロット・出荷履歴を参照できる
- 信頼性: 🔵 | 出典: ヒアリング確認済み（製番トレース機能）

---

## エピック6: 在庫管理

### US-025 在庫一覧・ロット確認
**As a** warehouse または admin,
**I want to** 在庫をロット別・倉庫別に確認したい,
**So that** 現在の在庫状況と各ロットの詳細情報を把握できる。

- 受け入れ基準:
  - ロット番号・製造日・有効期限・倉庫・残数量が一覧で確認できる
  - 倉庫・商品でフィルタリングできる
- 信頼性: 🔵 | 出典: PRD Phase 3

---

### US-026 倉庫間在庫移動
**As a** warehouse,
**I want to** 在庫を倉庫間で移動したい,
**So that** 出荷拠点や保管場所の都合に合わせて在庫を配置できる。

- 受け入れ基準:
  - 移動元倉庫・移動先倉庫・ロット・数量を指定して移動登録できる
  - 移動後に移動元の在庫が減算、移動先の在庫が加算される
- 信頼性: 🔵 | 出典: PRD Phase 3

---

### US-027 棚卸・在庫調整
**As a** warehouse または admin,
**I want to** 棚卸を実施して在庫数量を調整したい,
**So that** 実在庫とシステム上の在庫数量を一致させられる。

- 受け入れ基準:
  - 実棚卸数量を入力して在庫調整を登録できる
  - 調整前後の数量差異と調整理由を記録できる
  - 調整履歴が参照可能
- 信頼性: 🔵 | 出典: PRD Phase 3

---

## エピック7: 出荷管理

### US-028 出荷登録（ロット手動指定）
**As a** warehouse,
**I want to** ロットを手動で選択して出荷登録したい,
**So that** 出荷するロットを担当者が意図的に指定できる。

- 受け入れ基準:
  - 受注に対して出荷数量・倉庫・ロットを手動で指定できる
  - 指定ロットの在庫数量が出荷数量以上であることをチェックする
  - 出荷確定後に在庫数量が減算される
- 信頼性: 🔵 | 出典: PRD Phase 3 / ヒアリングQ4

---

### US-029 納品書PDF出力
**As a** warehouse または sales,
**I want to** 出荷登録後に納品書PDFをダウンロードしたい,
**So that** 顧客へ納品時に納品書を同梱できる。

- 受け入れ基準:
  - 出荷詳細ページから納品書PDFをダウンロードできる
  - PDFは日本語で出力される
- 信頼性: 🔵 | 出典: PRD Phase 5 / ヒアリングQ2

---

### US-030 出荷の返品・訂正処理
**As a** warehouse または admin,
**I want to** 出荷した商品の返品・訂正を登録したい,
**So that** 出荷ミスや顧客都合による返品を正確に記録できる。

- 受け入れ基準:
  - 出荷に対して返品数量・返品理由を登録できる
  - 返品登録後に在庫数量が加算される
  - 返品に伴う売上の訂正が可能
- 信頼性: 🔵 | 出典: PRD Phase 3

---

- 受け入れ基準:
  - 請求書詳細ページからPDFをダウンロードできる

## エピック9: 入金管理

### US-034 入金登録
**As a** admin,
**I want to** 得意先からの入金を登録したい,
**So that** 売掛金の回収状況を正確に管理できる。

- 受け入れ基準:
  - 得意先・入金日・入金額・入金方法を登録できる
  - 入金登録後に売掛金残高が更新される
- 信頼性: 🔵 | 出典: PRD Phase 4

---

### US-035 複数請求書への分割充当（消込）
**As a** admin,
**I want to** 1件の入金を複数の請求書に分割して充当したい,
**So that** 複数の未払い請求書がある場合も柔軟に消込できる。

- 受け入れ基準:

  - 充当金額の合計が入金額を超過しないことをチェックする
  - 充当後に各請求書の残高が更新される
- 信頼性: 🔵 | 出典: PRD Phase 4 / ヒアリングQ6

### US-036 入金消込の修正・取消
**As a** admin,
**I want to** 誤った入金消込を修正・取消したい,
**So that** 入力ミスが発生した場合も正確な状態に戻せる。

- 受け入れ基準:
  - 入金消込を取消して再充当できる
  - 取消時に売掛金残高と請求書残高が元に戻る
- 信頼性: 🔵 | 出典: ヒアリング確認済み（入金消込・分割充当）

## エピック10: レポート

### US-037 売上集計レポート（月次）
**As a** admin または sales,
**I want to** 月次の売上集計レポートを確認したい,
**So that** 月ごとの売上推移を把握して経営判断に活かせる。

- 受け入れ基準:
  - 月次の売上合計・受注件数・得意先別集計が表示される
  - 対象月を指定して絞り込みができる
  - グラフまたは表形式で表示される
- 信頼性: 🔵 | 出典: PRD Phase 5 / ヒアリングQ9

---

### US-038 売上集計レポート（得意先別・商品別）
**As a** admin または sales,
**I want to** 得意先別・商品別の売上集計を確認したい,
**So that** どの得意先・商品が収益に貢献しているかを把握できる。

- 受け入れ基準:
