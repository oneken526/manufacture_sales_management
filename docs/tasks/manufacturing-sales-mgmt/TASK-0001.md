## 実装詳細

### 1. Laravel Breeze インストール 🔵
**信頼性**: 🔵 *Laravel 公式ドキュメント・技術スタック定義より*

Blade スタックで認証スキャフォールドを生成する。

```bash
composer require laravel/breeze --dev
php artisan breeze:install blade
npm install && npm run build
```

- `php artisan breeze:install blade` で以下が自動生成される
  - `resources/views/auth/` 配下の認証ビュー群
  - `app/Http/Controllers/Auth/` 配下のコントローラ群
  - `routes/auth.php` の認証ルート
- `npm run build` で Vite により CSS/JS がビルドされる

**実装ファイル**: `resources/views/auth/`（Breeze が自動生成）、`routes/auth.php`

---

### 2. Spatie Laravel Permission インストール 🔵
**信頼性**: 🔵 *技術スタック定義（Spatie Permission 6.x）より*

4 ロール（admin / sales / manufacture / warehouse）による権限制御基盤を導入する。

```bash
composer require spatie/laravel-permission
php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider"
php artisan migrate
```

- `config/permission.php` が生成される（モデルクラスの参照先など設定可能）
- マイグレーションで `roles`、`permissions`、`model_has_roles`、`model_has_permissions`、`role_has_permissions` テーブルが作成される

**実装ファイル**: `config/permission.php`

---

### 3. barryvdh/laravel-dompdf インストール 🔵
**信頼性**: 🔵 *技術スタック定義より*

請求書・納品書などの PDF 出力に使用する。

```bash
composer require barryvdh/laravel-dompdf
php artisan vendor:publish --provider="Barryvdh\DomPDF\ServiceProvider"
```

- `config/dompdf.php` が生成される
- `Barryvdh\DomPDF\Facade\Pdf` ファサードを使用して PDF を生成する

**実装ファイル**: `config/dompdf.php`

---

### 4. jQuery 3.x 追加 🔵
**信頼性**: 🔵 *技術スタック定義（jQuery 3.x）より*

見積・受注画面の行追加／単価 Ajax 取得などに使用する。

```bash
npm install jquery
```

`resources/js/app.js` に以下を追加:

```javascript
// jQuery をグローバルに公開（Blade テンプレートからも使用可能にする）
import $ from 'jquery';
window.$ = $;
window.jQuery = $;
```

その後再ビルド:

```bash
npm run build
```

**実装ファイル**: `resources/js/app.js`

---

### 5. .env 設定確認 🔵
**信頼性**: 🔵 *技術スタック定義・EDGE-001（メール失敗時ロールバック）より*

```dotenv
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=manufacture_sales
DB_USERNAME=root
DB_PASSWORD=

MAIL_MAILER=log
```

- `MAIL_MAILER=log` に設定することでメール送信をログに記録し、開発中の意図しない外部送信を防ぐ
- メール送信失敗時のロールバック仕様（EDGE-001）はアプリケーション層で実装するが、基盤としてログドライバを使用する

**実装ファイル**: `.env`（`.env.example` も同様に更新）
