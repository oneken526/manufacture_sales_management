## タスク概要
製造業向け販売管理システムで使用する全業務テーブルのマイグレーションファイルを作成する。得意先・商品・受注・製造・在庫・出荷・請求・入金の各業務領域を網羅した 20 本のマイグレーションを `database/migrations/` に作成し、`php artisan migrate` で一括適用できる状態にする。全テーブルにソフトデリート（`deleted_at`）を設ける。

## 注意事項
- マイグレーションファイルの作成順序は外部キー制約の依存関係に従うこと
  - 例: `products` → `customer_special_prices`（`product_id` FK があるため）
- `customer_special_prices` は `customers` と `products` の両方が先に存在する必要がある
- `stock_movements` テーブルは履歴データのため `updated_at` を持たない（`created_at` のみ）
- CHECK 制約は `DB::statement()` で MySQL 8.0 の文法に従い記述する
- PHPUnit テストでは SQLite をインメモリで使用するため、CHECK 制約は SQLite 互換にするか、テスト側で確認方法を調整すること
- DIRECT タスクのため TDD プロセス不要
