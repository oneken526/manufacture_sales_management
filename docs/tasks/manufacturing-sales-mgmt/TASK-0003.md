## 実装詳細

### 1. User モデルに HasRoles・SoftDeletes トレイト追加 🔵
**信頼性**: 🔵 *Spatie Permission 6.x 公式ドキュメント・技術スタック定義より*

```php
// app/Models/User.php
<?php

declare(strict_types=1);

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\SoftDeletes;
use Illuminate\Foundation\Auth\User as Authenticatable;
use Illuminate\Notifications\Notifiable;
use Spatie\Permission\Traits\HasRoles;

class User extends Authenticatable
{
    use HasFactory, Notifiable, HasRoles, SoftDeletes;

    /**
     * 一括代入可能な属性
     */
    protected $fillable = [
        'name',
        'email',
        'password',
    ];

    /**
     * シリアライズ時に非表示にする属性
     */
    protected $hidden = [
        'password',
        'remember_token',
    ];

    /**
     * 属性のキャスト設定
     */
    protected function casts(): array
    {
        return [
            'email_verified_at' => 'datetime',
            'password'          => 'hashed',
        ];
    }
}
```

**実装ファイル**: `app/Models/User.php`

### 2. RoleSeeder 作成 🔵
**信頼性**: 🔵 *REQ-011（4ロール定義）より*

4 ロールを作成する。パーミッションはロール名で粗粒度に制御するため、本シーダーではロールのみ作成する。

```php
// database/seeders/RoleSeeder.php
<?php

declare(strict_types=1);

namespace Database\Seeders;

use Illuminate\Database\Seeder;
use Spatie\Permission\Models\Role;

class RoleSeeder extends Seeder
{
    /**
     * 4ロールを作成する
     * - admin: システム全体の管理（マスタ管理・ユーザー管理含む）
     * - sales: 見積・受注・得意先管理・レポート
     * - manufacture: 製造指示の参照・ステータス更新
     * - warehouse: 在庫管理・出荷処理
     */
    public function run(): void
    {
        $roles = ['admin', 'sales', 'manufacture', 'warehouse'];

        foreach ($roles as $roleName) {
            Role::firstOrCreate(['name' => $roleName]);
        }
    }
}
```

**実装ファイル**: `database/seeders/RoleSeeder.php`

### 3. AdminUserSeeder 作成 🔵
**信頼性**: 🔵 *REQ-011（初期管理者ユーザー）より*

初期管理者ユーザーを作成して `admin` ロールを付与する。

```php
// database/seeders/AdminUserSeeder.php
<?php

declare(strict_types=1);

namespace Database\Seeders;

use App\Models\User;
use Illuminate\Database\Seeder;
use Illuminate\Support\Facades\Hash;

class AdminUserSeeder extends Seeder
{
    /**
     * 初期管理者ユーザーを作成する
     * 本番環境では必ずパスワードを変更すること
     */
    public function run(): void
    {
        $admin = User::firstOrCreate(
            ['email' => 'admin@example.com'],
            [
                'name'     => '管理者',
                'password' => Hash::make('password'),
            ]
        );

        // admin ロールを付与（Spatie Permission）
        $admin->assignRole('admin');
    }
}
```

**実装ファイル**: `database/seeders/AdminUserSeeder.php`

### 4. DatabaseSeeder 更新 🔵
**信頼性**: 🔵 *Laravel シーダー設計より*

`RoleSeeder` を先に実行し、その後 `AdminUserSeeder` を実行することでロールが存在する状態でユーザーにロールを付与できる。

```php
// database/seeders/DatabaseSeeder.php
<?php

declare(strict_types=1);

namespace Database\Seeders;

use Illuminate\Database\Seeder;

class DatabaseSeeder extends Seeder
{
    /**
     * シーダーを実行する
     * 実行順序: RoleSeeder → AdminUserSeeder（依存関係あり）
     */
    public function run(): void
    {
        $this->call([
            RoleSeeder::class,
            AdminUserSeeder::class,
        ]);
    }
}
```

**実装ファイル**: `database/seeders/DatabaseSeeder.php`

## 注意事項
- `RoleSeeder` は `firstOrCreate` を使用して冪等性を確保すること（複数回実行しても重複しない）
- `AdminUserSeeder` も `firstOrCreate` を使用すること
- Spatie Permission のキャッシュがある場合は `php artisan permission:cache-reset` で初期化が必要
- テスト環境では `RefreshDatabase` トレイトを使用し、各テスト後にDBをリセットすること
- 本番環境では `admin@example.com` / `password` を必ず変更すること（セキュリティ注意事項）
- DIRECT タスクのため TDD プロセス不要
