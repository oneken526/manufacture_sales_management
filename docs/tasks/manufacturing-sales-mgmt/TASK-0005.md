## タスク概要
Laravel Breeze が生成した認証ビュー（ログイン・パスワード忘れ・パスワードリセット）を Tailwind CSS 4.0 でシステム名に合わせてカスタマイズする。ログイン後のリダイレクト先をロール別に変更し（admin/sales → `/dashboard`、manufacture → `/manufacture-orders`、warehouse → `/shipments`）、ログアウト後は `/login` に遷移する。

### 4. ログイン後リダイレクト先をロール別に変更 🔵
**信頼性**: 🔵 *REQ-011（ロール別リダイレクト）より*

Breeze の `AuthenticatedSessionController` を拡張してロール別にリダイレクト先を変更する。

```php
// app/Http/Controllers/Auth/AuthenticatedSessionController.php
<?php

declare(strict_types=1);

namespace App\Http\Controllers\Auth;

use App\Http\Controllers\Controller;
use App\Http\Requests\Auth\LoginRequest;
use Illuminate\Http\RedirectResponse;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Auth;
use Illuminate\View\View;

class AuthenticatedSessionController extends Controller
{
    /**
     * ログインフォームを表示する
     */
    public function create(): View
    {
        return view('auth.login');
    }

    /**
     * ログイン処理を行い、ロール別にリダイレクトする
     */
    public function store(LoginRequest $request): RedirectResponse
    {
        $request->authenticate();
        $request->session()->regenerate();

        $user = Auth::user();

        // ロール別リダイレクト先を決定する
        $redirectTo = match (true) {
            $user->hasRole('manufacture') => route('manufacture-orders.index'),
            $user->hasRole('warehouse')   => route('shipments.index'),
            default                       => route('dashboard'), // admin, sales
        };

        return redirect()->intended($redirectTo);
    }

    /**
     * ログアウト処理を行い、ログイン画面にリダイレクトする
     */
    public function destroy(Request $request): RedirectResponse
    {
        Auth::guard('web')->logout();

        $request->session()->invalidate();
        $request->session()->regenerateToken();

        return redirect()->route('login');
    }
}
```

**実装ファイル**: `app/Http/Controllers/Auth/AuthenticatedSessionController.php`

## 注意事項
- Breeze が生成した `AuthenticatedSessionController` を上書きする形でロール別リダイレクトを追加すること
- `redirect()->intended($redirectTo)` を使用することで、認証前にアクセスしようとした URL へのリダイレクトも正しく動作する
- テスト環境では `Mail::fake()` または `MAIL_MAILER=log` を使用してメール送信を検証すること
- `manufacture-orders` と `shipments` のルートは後続タスクで実装されるため、テストでは存在しない場合の 404 を考慮すること
