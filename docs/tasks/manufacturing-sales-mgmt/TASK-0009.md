## タスク概要

システムユーザーの管理（登録・編集・ロール付与・論理削除）を提供する `UserController` を実装する。admin ロールのみが操作可能であり、ユーザー作成時にパスワードを設定し、Spatie Permission でロールを付与する。自身のアカウントは削除不可とする。

## 実装詳細

### 1. UserController 🔵

**信頼性**: 🔵 *Webルート仕様・要件定義 REQ-010 より*

**実装ファイル**: `app/Http/Controllers/UserController.php`

```php
public function store(UserRequest $request) {
    $user = User::create([
        'name'     => $request->name,
        'email'    => $request->email,
        'password' => Hash::make($request->password),
    ]);
    $user->syncRoles([$request->role]);
    return redirect()->route('users.index')->with('success', 'ユーザーを作成しました');
}

public function destroy(User $user) {
    if ($user->id === auth()->id()) {
        return back()->with('error', '自身のアカウントは削除できません');
    }
    $user->delete();
    return redirect()->route('users.index')->with('success', 'ユーザーを削除しました');
}
```

### 2. バリデーション 🔵

**信頼性**: 🔵 *要件定義 REQ-010 より*

**実装ファイル**: `app/Http/Requests/UserRequest.php`

- `name`: required, max:255
- `email`: required, email, unique:users（更新時は自身除外）
- `password`: required（作成時）, min:8（confirmed）
- `role`: required, in:admin,sales,manufacture,warehouse

### 3. ルート設定 🔵

**信頼性**: 🔵 *Webルート仕様より*

```php
Route::middleware(['auth', 'role:admin'])->group(function () {
    Route::resource('users', UserController::class)->except(['show']);
});
```
