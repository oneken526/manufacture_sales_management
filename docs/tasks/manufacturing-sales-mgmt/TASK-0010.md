## タスク概要

得意先ごとの商品別特別単価を管理する画面と Ajax エンドポイントを実装する。得意先詳細画面から特別単価の追加・編集・削除が可能であり、見積・受注の明細行で商品を選択した際に Ajax で特別単価を自動反映する（REQ-020）。特別単価がない場合は商品マスタの標準単価を返す。

- **後続タスク**: TASK-0013（QuotationService）, TASK-0015（見積作成・編集画面）

## 完了条件

- [ ] 🔵 `GET /customers/{id}/special-prices` で得意先の特別単価一覧が表示される
- [ ] 🔵 `POST /customers/{id}/special-prices` で特別単価を登録できる（商品ごと 1 件のみ）
- [ ] 🔵 `PUT /customers/{id}/special-prices/{spId}` で単価を更新できる
- [ ] 🔵 `DELETE /customers/{id}/special-prices/{spId}` で特別単価を削除できる
- [ ] 🔵 `GET /api/customers/{id}/unit-price?product_id={pid}` で適用単価（特別 or 標準）を JSON で返す（REQ-020）
- [ ] 🔵 同一得意先・同一商品の特別単価が重複登録されるとバリデーションエラーになる

## 実装詳細

### 1. CustomerSpecialPriceController 🔵

**信頼性**: 🔵 *Webルート仕様・要件定義 REQ-006 より*

**実装ファイル**: `app/Http/Controllers/CustomerSpecialPriceController.php`

```php
public function store(Request $request, Customer $customer) {
    $request->validate([
        'product_id' => ['required', 'exists:products,id',
            Rule::unique('customer_special_prices')->where(fn($q) =>
                $q->where('customer_id', $customer->id)->whereNull('deleted_at')
            )],
        'unit_price' => ['required', 'numeric', 'min:0'],
    ]);
    $customer->specialPrices()->create($request->only('product_id', 'unit_price'));
    return back()->with('success', '特別単価を登録しました');
}
```

### 2. 単価取得 Ajax エンドポイント 🔵

**信頼性**: 🔵 *要件定義 REQ-020・Webルート仕様より*

**実装ファイル**: `app/Http/Controllers/Api/UnitPriceController.php`

```php
public function show(Customer $customer, Request $request) {
    $product = Product::findOrFail($request->product_id);
    $special = $customer->specialPrices()
        ->where('product_id', $product->id)->first();
    return response()->json([
        'unit_price' => $special ? $special->unit_price : $product->unit_price,
        'is_special' => (bool) $special,
    ]);
}
```

### テストケース1: 特別単価の優先返却 🔵
