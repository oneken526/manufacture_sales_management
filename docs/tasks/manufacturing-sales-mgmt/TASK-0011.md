## タスク概要
見積番号・受注番号・製造指示番号・請求書番号・出荷番号など、各業務ドキュメントに一意の採番を行うサービスクラスを実装する。採番形式は `{PREFIX}{YYYYMM}-{3桁連番}` とし、DBロックにより並行処理での重複を防止する。

## 実装詳細

### 1. NumberingService 実装 🔵
**信頼性**: 🔵 *採番形式仕様（{PREFIX}{YYYYMM}-{3桁連番}）より*

**実装ファイル**: `app/Services/NumberingService.php`

- `generate(string $prefix): string` メソッドを実装する
- 引数 `$prefix` は以下の値を受け付ける
  - `Q` : 見積（Quotation）
  - `O` : 受注（Order）
  - `MO`: 製造指示（ManufactureOrder）
  - `INV`: 請求（Invoice）
  - `SH` : 出荷（Shipment）
- ロジック概要:
  1. 現在の年月（`YYYYMM`）を取得する
  2. 対象テーブルから `{prefix}{YYYYMM}-` で始まるレコードを `lockForUpdate()` で取得する
  3. 最大連番 + 1 を3桁ゼロ埋めで返す（初回は `001`）
- DBロック: `DB::transaction()` 内で `lockForUpdate()` を使用し、並行生成での重複を防止する 🔵

```php
<?php

declare(strict_types=1);

namespace App\Services;

use Illuminate\Support\Facades\DB;

class NumberingService
{
    /**
     * プレフィックスと現在年月に基づいて採番した番号を返す。
     * 形式: {PREFIX}{YYYYMM}-{3桁連番} 例: Q202604-001
     */
    public function generate(string $prefix): string
    {
        return DB::transaction(function () use ($prefix): string {
            $yearMonth = now()->format('Ym');
            $like = $prefix . $yearMonth . '-%';
            $table = $this->resolveTable($prefix);

            $max = DB::table($table)
                ->where($this->resolveColumn($prefix), 'like', $like)
                ->lockForUpdate()
                ->max($this->resolveColumn($prefix));

            $next = 1;
            if ($max !== null) {
                $parts = explode('-', $max);
                $next = (int) end($parts) + 1;
            }

            return $prefix . $yearMonth . '-' . str_pad((string) $next, 3, '0', STR_PAD_LEFT);
        });
    }

    private function resolveTable(string $prefix): string
    {
        return match ($prefix) {
            'Q'   => 'quotations',
            'O'   => 'orders',
            'MO'  => 'manufacture_orders',
            'INV' => 'invoices',
            'SH'  => 'shipments',
            default => throw new \InvalidArgumentException("Unknown prefix: {$prefix}"),
        };
    }

    private function resolveColumn(string $prefix): string
    {
        return match ($prefix) {
            'Q'   => 'quotation_number',
            'O'   => 'order_number',
            'MO'  => 'manufacture_number',
            'INV' => 'invoice_number',
            'SH'  => 'shipment_number',
            default => throw new \InvalidArgumentException("Unknown prefix: {$prefix}"),
        };
    }
}
```

## テスト要件（Feature テスト）

**テストファイル**: `tests/Feature/Services/NumberingServiceTest.php`

### テストケース1: 最初の見積採番 🔵
**信頼性**: 🔵 *採番形式仕様より*
**Given**: quotations テーブルにレコードが存在しない
**When**: `NumberingService::generate('Q')` を呼び出す
**Then**: `Q{YYYYMM}-001` 形式の文字列が返る

### テストケース2: 連番インクリメント 🔵
**信頼性**: 🔵 *採番仕様より*
**Given**: 当月の見積が2件登録済み
**When**: `generate('Q')` を呼び出す
**Then**: `Q{YYYYMM}-003` が返る

### テストケース3: プレフィックス独立性 🔵
**信頼性**: 🔵 *採番仕様より*
**Given**: `generate('Q')` で見積番号が1件採番済み
**When**: `generate('O')` を呼び出す
**Then**: `O{YYYYMM}-001` が返る（Q の連番に影響されない）

### テストケース4: 並行生成での重複なし 🔵
**信頼性**: 🔵 *lockForUpdate() 使用・競合防止仕様より*
**Given**: quotations テーブルにレコードが存在しない
**When**: `generate('Q')` を複数回連続で呼び出す
**Then**: 生成された番号はすべて一意である

## 注意事項
- PHP 8.3 の `declare(strict_types=1)` および型宣言を使用すること
- `DB::transaction()` と `lockForUpdate()` は必ずセットで使用し、採番の重複を防止すること
- テスト内では `RefreshDatabase` トレイトを使用してテーブルをリセットすること
- 各テーブルの採番カラム名はテーブルごとに異なるため `resolveColumn()` で分岐すること
- 翌月以降は連番がリセットされる（年月が変わると `001` から再開）
