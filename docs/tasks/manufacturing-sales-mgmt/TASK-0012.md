## 実装詳細

### 1. ApprovalRequestMail（承認申請メール） 🔵
**信頼性**: 🔵 *REQ-021（admin への承認申請通知）より*

**実装ファイル**: `app/Mail/ApprovalRequestMail.php`

- 承認申請時に admin ロールを持つ全ユーザーへ送信する
- コンストラクタ引数:
  - `$document`: `Quotation` または `Order` モデルインスタンス
  - `$requester`: 申請を行った `User` モデルインスタンス
- 件名: `【承認申請】{ドキュメント番号}`
  - 例: `【承認申請】Q202604-001`
- ビュー: `resources/views/emails/approval-request.blade.php`
- メール本文に含める情報:
  - ドキュメント番号
  - 申請者名
  - 申請日時
  - 得意先名
  - 承認画面へのリンク

```php
<?php

declare(strict_types=1);

namespace App\Mail;

use Illuminate\Bus\Queueable;
use Illuminate\Mail\Mailable;
use Illuminate\Mail\Mailables\Content;
use Illuminate\Mail\Mailables\Envelope;
use Illuminate\Queue\SerializesModels;

class ApprovalRequestMail extends Mailable
{
    use Queueable, SerializesModels;

    public function __construct(
        public readonly mixed $document,
        public readonly \App\Models\User $requester,
    ) {}

    public function envelope(): Envelope
    {
        $number = $this->resolveDocumentNumber();
        return new Envelope(subject: "【承認申請】{$number}");
    }

    public function content(): Content
    {
        return new Content(view: 'emails.approval-request');
    }

    private function resolveDocumentNumber(): string
    {
        return $this->document->quotation_number
            ?? $this->document->order_number
            ?? '';
    }
}
```

### 4. メール送信の同期処理（EDGE-001） 🔵
**信頼性**: 🔵 *EDGE-001（メール失敗時ロールバック）より*

- `Mail::to($admin)->send(new ApprovalRequestMail(...))` を使用する（キューは使わない）
- メール送信は必ず `DB::transaction()` ブロック内で行う
- `Mail::send()` が例外をスローした場合、トランザクションごとロールバックされる
- 呼び出し元（QuotationService / OrderService）でこの仕様を遵守する

```php
// 呼び出し例（サービスクラス内）
DB::transaction(function () use ($quotation, $requester) {
    $quotation->update(['status' => 'pending']);

    $admins = User::role('admin')->get();
    foreach ($admins as $admin) {
        Mail::to($admin)->send(new ApprovalRequestMail($quotation, $requester));
    }
    // メール送信失敗時は例外がスローされ、トランザクションがロールバックされる
});
```

## 注意事項
- DIRECT タスクのため TDD プロセス不要。`direct-setup` → `direct-verify` で実施する
- PHP 8.3 の `declare(strict_types=1)` および型宣言を使用すること
- メール送信は必ず同期処理（`send()`）とし、`queue()` や `later()` は使用しないこと（EDGE-001）
- `Mail::fake()` を使用したテストでは実際のメール送信は行われないため、ログへの記録テストと分けて実施すること
- Quotation と Order の両方を `$document` として受け取れるよう、ドキュメント番号の解決は `resolveDocumentNumber()` で行うこと
