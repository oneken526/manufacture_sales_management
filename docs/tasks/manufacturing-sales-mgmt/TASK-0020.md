## タスク概要

受注書の承認ワークフロー（申請→承認/却下）を実装する。見積承認と同様のパターン（`OrderService::requestApproval` / `approve` / `reject`）を適用する。承認時は `ManufactureOrderService::createFromOrder()` を呼んで製造指示を自動発行し（REQ-023）、失敗時はロールバック（EDGE-007）。メール送信失敗時もロールバック（EDGE-001）。

## 実装詳細

### 1. OrderController（承認アクション） 🔵

**信頼性**: 🔵 *Webルート仕様・要件定義 REQ-017 より*

**実装ファイル**: `app/Http/Controllers/OrderController.php`（追記）

```php
public function requestApproval(Order $order) {
    $this->authorize('requestApproval', $order);
    $this->orderService->requestApproval($order, auth()->user());
    return back()->with('success', '承認申請しました');
}

public function approve(Order $order) {
    $this->authorize('approve', $order); // admin のみ
    $this->orderService->approve($order, auth()->user());
    return back()->with('success', '受注を承認しました（製造指示を発行しました）');
}

public function reject(Order $order) {
    $this->authorize('approve', $order); // admin のみ
    $this->orderService->reject($order, auth()->user(), request('reason'));
    return back()->with('success', '受注を却下しました');
}
```

### 2. ルート設定 🔵

```php
Route::middleware(['auth', 'role:admin'])->group(function () {
    Route::post('/orders/{order}/approve', [OrderController::class, 'approve']);

});
```

## 単体テスト要件

### テストケース1: 受注承認で製造指示が自動発行される 🔵

**Given**: status=pending の受注（明細 2 行）
**When**: `OrderService::approve()` を呼ぶ
**Then**: ManufactureOrder が 1 件生成される（1受注 = 1製造指示）

### テストケース2: 製造指示発行失敗でロールバック（EDGE-007） 🔵

**Given**: status=pending の受注
**When**: `ManufactureOrderService::createFromOrder()` が例外をスローする
**Then**: 受注は pending のまま、製造指示も作成されない

### テストケース3: sales が approve すると 403 🔵

**Given**: sales ユーザー
**When**: POST /orders/{id}/approve
**Then**: 403 Forbidden が返る
