## タスク概要
製造指示（ManufactureOrder）の業務ロジックを担うサービスクラスを実装する。NumberingService による製番採番（MO{YYYYMM}-001 形式）、ステータス遷移（pending → in_progress → completed / cancelled）、製造完了時の在庫入庫連動（StockService::inbound() 呼び出し）を提供する。

## 依存タスク
- **前提タスク**: TASK-0020
- **後続タスク**: TASK-0022, TASK-0023

## 完了条件
- [ ] 🔵 `ManufactureOrderService::createFromOrder()` が製番を MO{YYYYMM}-001 形式で採番し、ステータス pending の ManufactureOrder を作成する
- [ ] 🔵 `start()` により pending → in_progress へ遷移し、`started_at` が記録される
- [ ] 🔵 `complete()` により in_progress → completed へ遷移し、`completed_at` が記録される
- [ ] 🔵 `complete()` 実行時に `StockService::inbound()` が呼び出され在庫入庫が行われる
- [ ] 🔵 `complete()` はトランザクション内で実行される
- [ ] 🟡 `complete()` を completed 状態の製造指示に対して実行した場合、スキップして警告を返す（EDGE-010）
- [ ] 🔵 `cancel()` は admin ロールのみ実行でき、pending または in_progress の製造指示のみキャンセル可能
- [ ] 🔵 同一 order_id に対して `createFromOrder()` を2回呼び出した場合、2件目は unique 制約エラーになる
- [ ] 🔵 `php artisan test` で全テストがパスする

## 実装詳細

### 1. ManufactureOrderService 🔵
**信頼性**: 🔵 *REQ-027, REQ-028, REQ-035, REQ-039 より*

**ファイル**: `app/Services/ManufactureOrderService.php`

#### createFromOrder(Order $order): ManufactureOrder 🔵
**信頼性**: 🔵 *REQ-039（1受注=1製造指示）より*

- `NumberingService::generate('MO')` を呼び出し、`MO{YYYYMM}-001` 形式の製番を採番する
- `manufacture_orders` テーブルに `order_id`（unique）、`manufacture_number`、ステータス `pending` で保存する
- `order_id` には unique 制約を設定済みのため、同一受注への重複登録は DB レベルで防止される

#### start(ManufactureOrder $mo): void 🔵
**信頼性**: 🔵 *REQ-027, REQ-028 より*

- ステータスが `pending` であることを確認する（それ以外は例外をスロー）
- `status` を `in_progress` に更新し、`started_at` に現在時刻を記録する

#### complete(ManufactureOrder $mo, array $lotData): void 🟡
**信頼性**: 🟡 *REQ-035（完了時の在庫入庫）は明確。EDGE-010（completed スキップ）は設計メモより*

- `status` が `completed` の場合はスキップしてログ警告を記録する（EDGE-010）
- `status` が `in_progress` でない場合は例外をスロー
- DB::transaction() 内で以下を実行する
  - `status` を `completed` に更新し、`completed_at` に現在時刻を記録する
  - `StockService::inbound($lotData, $mo, $warehouse)` を呼び出して在庫入庫を行う
  - `lotData` キー: `lot_number`, `manufactured_date`, `expiry_date`, `quantity`, `warehouse_id`

#### cancel(ManufactureOrder $mo, User $admin): void 🔵
**信頼性**: 🔵 *REQ-028（キャンセル権限）より*

- `$admin->hasRole('admin')` が false の場合は `AuthorizationException` をスロー
- `status` が `pending` または `in_progress` でない場合は例外をスロー
- `status` を `cancelled` に更新し、`cancelled_at` に現在時刻を記録する

**実装ファイル**: `app/Services/ManufactureOrderService.php`

### テストケース一覧

#### TC-0021-01: createFromOrder → 製番が MO{YYYYMM}-001 形式 🔵
**信頼性**: 🔵 *REQ-039（製番採番ルール）より*

```
前提: 受注（Order）が存在する
操作: ManufactureOrderService::createFromOrder($order) を実行する
期待: manufacture_number が "MO" + 年月6桁 + "-001" 形式である
      status が "pending" である
      order_id が $order->id である
```

#### TC-0021-02: start（pending）→ in_progress・started_at 記録 🔵
**信頼性**: 🔵 *REQ-027, REQ-028 より*

```
前提: status=pending の ManufactureOrder が存在する
操作: ManufactureOrderService::start($mo) を実行する
期待: status が "in_progress" に変更されている
      started_at が null でない
```

#### TC-0021-03: complete（in_progress）→ completed・在庫入庫実行 🔵
**信頼性**: 🔵 *REQ-035（完了時の在庫入庫）より*

```
前提: status=in_progress の ManufactureOrder が存在する
      有効な lotData（lot_number, manufactured_date, expiry_date, quantity, warehouse_id）を準備する
操作: ManufactureOrderService::complete($mo, $lotData) を実行する
期待: status が "completed" に変更されている
      completed_at が null でない
      stock_lots テーブルに1件追加されている
      stock_quantities テーブルに1件追加されている
      stock_movements テーブルに movement_type=inbound の1件が追加されている
```

#### TC-0021-04: complete（completed 状態の再実行）→ スキップ・警告（EDGE-010） 🟡
**信頼性**: 🟡 *EDGE-010（設計メモ）より*

```
前提: status=completed の ManufactureOrder が存在する
操作: ManufactureOrderService::complete($mo, $lotData) を再度実行する
期待: 例外がスローされない（スキップされる）
      stock_lots テーブルに追加がない（在庫入庫が重複しない）
```

#### TC-0021-05: cancel（manufacture ロール）→ 403 🔵
**信頼性**: 🔵 *REQ-028（admin のみキャンセル可）より*

```
前提: manufacture ロールのユーザーが存在する
      status=pending の ManufactureOrder が存在する
操作: ManufactureOrderService::cancel($mo, $manufactureUser) を実行する
期待: AuthorizationException がスローされる
      status が変化していない
```

#### TC-0021-06: createFromOrder を同一 order_id で2回 → unique 制約エラー 🔵
**信頼性**: 🔵 *REQ-039（1受注=1製造指示の unique 制約）より*

```
前提: 受注（Order）が存在する
操作: ManufactureOrderService::createFromOrder($order) を2回実行する
期待: 2回目で QueryException（unique 制約違反）がスローされる
      manufacture_orders テーブルの件数が1件のまま
```
