## タスク概要

在庫管理の 2 層データモデル（`stock_lots` + `stock_quantities`）に対応するモデルと `StockService` を実装する。`StockService` は入庫（製造完了時）・出庫（出荷時）・倉庫間移動・在庫調整の 4 操作を提供し、全操作は `stock_movements` に履歴を記録する。

## 実装詳細

### 1. StockLot モデル 🔵

**信頼性**: 🔵 *DBスキーマ・要件定義 REQ-029 より*

**実装ファイル**: `app/Models/StockLot.php`

```php
class StockLot extends Model {
    use SoftDeletes, HasFactory;
    protected $fillable = [
        'product_id', 'manufacture_order_id', 'lot_number',
        'manufactured_date', 'expiry_date', 'notes',
    ];
    public function stockQuantities()  { return $this->hasMany(StockQuantity::class, 'lot_id'); }
    public function product()          { return $this->belongsTo(Product::class); }
    public function manufactureOrder() { return $this->belongsTo(ManufactureOrder::class); }
}
```

### 3. StockService::inbound 🔵

**信頼性**: 🔵 *要件定義 REQ-035・dataflow.md より*

**実装ファイル**: `app/Services/StockService.php`

```php
public function inbound(ManufactureOrder $mo, array $lotData, int $warehouseId, int $qty): StockLot {
    return DB::transaction(function () use ($mo, $lotData, $warehouseId, $qty) {
        $lot = StockLot::create(array_merge($lotData, [
            'product_id'          => $mo->product_id,
            'manufacture_order_id'=> $mo->id,
        ]));
        StockQuantity::create(['lot_id' => $lot->id, 'warehouse_id' => $warehouseId,
                               'quantity' => $qty]);
        StockMovement::create(['lot_id' => $lot->id, 'warehouse_id' => $warehouseId,
                               'type' => 'inbound', 'quantity' => $qty,
                               'manufacture_order_id' => $mo->id]);
        return $lot;
    });
}
```

### 4. StockService::outbound 🔵

**信頼性**: 🔵 *要件定義 REQ-033・EDGE-002 より*

```php
public function outbound(StockLot $lot, int $warehouseId, int $qty, ?int $shipmentId = null): void {
    DB::transaction(function () use ($lot, $warehouseId, $qty, $shipmentId) {
        $sq = StockQuantity::where(['lot_id' => $lot->id, 'warehouse_id' => $warehouseId])
            ->lockForUpdate()->firstOrFail();
        if ($sq->quantity < $qty) {
            throw new InsufficientStockException("在庫不足: ロット {$lot->lot_number}");
        }
        $sq->decrement('quantity', $qty);
        StockMovement::create(['lot_id' => $lot->id, 'warehouse_id' => $warehouseId,
                               'type' => 'outbound', 'quantity' => $qty,
                               'shipment_id' => $shipmentId]);
    });
}
```

### テストケース2: 在庫不足で outbound が例外をスロー（EDGE-002） 🔵

**Given**: 在庫数量=10 のロット
**When**: `StockService::outbound($lot, $warehouseId, 20)` を呼ぶ
**Then**: `InsufficientStockException` がスローされ DB は変更されない

**Given**: 倉庫A に qty=100 のロット
**When**: `StockService::transfer($lot, $warehouseA, $warehouseB, 30)` を呼ぶ
**Then**: 倉庫A=70、倉庫B=30 になり StockMovement(type=transfer) が記録される
