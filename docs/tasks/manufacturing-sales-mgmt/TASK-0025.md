## タスク概要

在庫ロットを別倉庫へ移動する画面を実装する。移動元ロット・移動元倉庫・移動先倉庫・数量を入力し、`StockService::transfer()` を呼ぶ。数量が移動元在庫を超える場合はバリデーションエラー。アクセスは admin / warehouse ロールに限定する（REQ-031）。

## 依存タスク

- **前提タスク**: TASK-0023（StockService）, TASK-0024（在庫一覧）
- **後続タスク**: なし

## 完了条件

- [ ] 🔵 `GET /stocks/transfer` で移動フォームが表示される（admin / warehouse のみ）
- [ ] 🔵 `POST /stocks/transfer` で `StockService::transfer()` が呼ばれ在庫が移動する（REQ-031）

- [ ] 🔵 移動元と移動先が同一倉庫の場合はバリデーションエラーが返る

## 実装詳細

### 1. StockController::transfer 🔵

**信頼性**: 🔵 *Webルート仕様・要件定義 REQ-031 より*

**実装ファイル**: `app/Http/Controllers/StockController.php`（追記）

```php
public function transfer(StockTransferRequest $request) {
    $lot = StockLot::findOrFail($request->lot_id);
    $this->stockService->transfer(
        $lot,
        $request->from_warehouse_id,
        $request->to_warehouse_id,
        $request->quantity
    );
    return redirect()->route('stocks.index')->with('success', '倉庫間移動を実行しました');
}
```

### 2. バリデーション 🔵

**信頼性**: 🔵 *要件定義 REQ-031 より*

**実装ファイル**: `app/Http/Requests/StockTransferRequest.php`

- `lot_id`: required, exists:stock_lots,id
- `from_warehouse_id`: required, exists:warehouses,id
- `to_warehouse_id`: required, exists:warehouses,id, different:from_warehouse_id
- `quantity`: required, integer, min:1
