## タスク概要

出荷管理の中核となる `Shipment` / `ShipmentDetail` モデルと `ShipmentService` を実装する。`ShipmentService::create()` は出荷番号採番・ロット手動引当・在庫減算・受注残更新を担う。`ShipmentService::return_()` は在庫加算・受注残復元の返品処理を担う。在庫不足時はロールバック（EDGE-002）。

## 実装詳細

### 1. Shipment モデル 🔵

**信頼性**: 🔵 *DBスキーマ・要件定義 REQ-033 より*

**実装ファイル**: `app/Models/Shipment.php`

```php
class Shipment extends Model {
    use SoftDeletes, HasFactory;
    protected $fillable = [
        'order_id', 'warehouse_id', 'shipment_number',
        'status', 'shipped_date', 'notes',
    ];
    public function order()    { return $this->belongsTo(Order::class); }
    public function details()  { return $this->hasMany(ShipmentDetail::class); }
    public function warehouse(){ return $this->belongsTo(Warehouse::class); }
}
```

### 2. ShipmentService::create 🔵

**信頼性**: 🔵 *要件定義 REQ-033・REQ-036・EDGE-002・dataflow.md より*

**実装ファイル**: `app/Services/ShipmentService.php`

```php
public function create(Order $order, int $warehouseId, array $details, User $actor): Shipment {
    return DB::transaction(function () use ($order, $warehouseId, $details, $actor) {
        $shipment = Shipment::create([
            'order_id'         => $order->id,
            'warehouse_id'     => $warehouseId,
            'shipment_number'  => $this->numberingService->generate('SH'),
            'status'           => 'draft',
            'created_by'       => $actor->id,
        ]);
        foreach ($details as $detail) {
            $lot = StockLot::findOrFail($detail['lot_id']);
            // 在庫不足なら EDGE-002 でロールバック
            $this->stockService->outbound($lot, $warehouseId, $detail['quantity'], $shipment->id);
            $shipment->details()->create($detail);
            // 受注残を更新
            $orderDetail = OrderDetail::findOrFail($detail['order_detail_id']);
            $orderDetail->increment('shipped_quantity', $detail['quantity']);
        }
        return $shipment;
    });
}
```

### 3. ShipmentService::return_ 🔵

**信頼性**: 🔵 *要件定義 REQ-034・REQ-037 より*

```php
public function return_(Shipment $shipment, User $actor): void {
    DB::transaction(function () use ($shipment, $actor) {
        foreach ($shipment->details as $detail) {
            $this->stockService->inbound(/* ロット情報から入庫 */);
            $detail->orderDetail->decrement('shipped_quantity', $detail->quantity);
        }
        $shipment->update(['status' => 'returned', 'returned_by' => $actor->id,
                           'returned_at' => now()]);
    });
}
```

## 単体テスト要件

### テストケース1: 出荷作成で在庫が減算され受注残が更新される 🔵

**Given**: 在庫 qty=50 のロット、受注 qty=30
**When**: `ShipmentService::create()` で qty=20 を出荷
**Then**: 在庫 qty=30、OrderDetail.shipped_quantity=20 になる

### テストケース2: 在庫不足でロールバック（EDGE-002） 🔵

**Given**: 在庫 qty=5 のロット
**When**: `ShipmentService::create()` で qty=10 を出荷しようとする
**Then**: `InsufficientStockException` がスローされ、出荷・在庫変更ともにロールバックされる
