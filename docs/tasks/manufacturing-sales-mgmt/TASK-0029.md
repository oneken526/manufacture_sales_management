## タスク概要

出荷の確定（draft → confirmed）、返品処理（confirmed → returned）、および納品書 PDF 出力を実装する。出荷確定後に月次請求書生成の対象となる（`InvoiceService` の前提）。返品時は `ShipmentService::return_()` で在庫を加算し受注残を復元する（REQ-037）。納品書 PDF は dompdf で出力する（REQ-047）。

## 実装詳細

### 1. ShipmentController::confirm 🔵

**信頼性**: 🔵 *Webルート仕様・要件定義 REQ-033 より*

**実装ファイル**: `app/Http/Controllers/ShipmentController.php`（追記）

```php
public function confirm(Shipment $shipment) {
    if ($shipment->status !== 'draft') {
        return back()->with('error', '確定できる状態ではありません');
    }
    $shipment->update(['status' => 'confirmed', 'shipped_date' => now()]);
    return back()->with('success', '出荷を確定しました');
}

public function return_(Shipment $shipment) {
    $this->shipmentService->return_($shipment, auth()->user());
    return back()->with('success', '返品処理を実行しました');
}
```

### 2. 納品書 PDF 出力 🔵

**信頼性**: 🔵 *要件定義 REQ-047 より*

```php
public function pdf(Shipment $shipment) {
    $pdf = \Barryvdh\DomPDF\Facade\Pdf::loadView('pdf.delivery', compact('shipment'));
    $pdf->setPaper('A4');
    return $pdf->download("delivery-{$shipment->shipment_number}.pdf");
}
```

**Blade テンプレート**: `resources/views/pdf/delivery.blade.php`
- 得意先名・出荷番号・出荷日・明細（商品名・ロット番号・数量）
