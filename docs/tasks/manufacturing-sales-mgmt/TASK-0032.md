## タスク概要

入金管理の中核となる `Payment` / `PaymentAllocation` モデルと `PaymentService` を実装する。`PaymentService::register()` は入金登録と複数請求書への分割充当（消込）を担い、`unallocated_amount`（未充当残高）を管理する。`PaymentService::cancel()` は消込取消処理を提供する。充当額が請求残高を超える場合はエラー（EDGE-005）。

### 1. Payment モデル 🔵

**信頼性**: 🔵 *DBスキーマ・要件定義 REQ-042 より*

**実装ファイル**: `app/Models/Payment.php`

```php
class Payment extends Model {
    use SoftDeletes, HasFactory;
    protected $fillable = [
        'customer_id', 'payment_date', 'amount',
        'unallocated_amount', 'payment_method', 'notes',
    ];
    public function customer()    { return $this->belongsTo(Customer::class); }
    public function allocations() { return $this->hasMany(PaymentAllocation::class); }
}
```

### 3. PaymentService::register 🔵

**信頼性**: 🔵 *要件定義 REQ-042・REQ-043・EDGE-005・dataflow.md より*

**実装ファイル**: `app/Services/PaymentService.php`

```php
public function register(Customer $customer, array $data, array $allocations, User $actor): Payment {
    return DB::transaction(function () use ($customer, $data, $allocations, $actor) {
        $payment = Payment::create(array_merge($data, [
            'customer_id'        => $customer->id,
            'unallocated_amount' => $data['amount'],
            'created_by'         => $actor->id,
        ]));

        foreach ($allocations as $alloc) {
            $invoice = Invoice::lockForUpdate()->findOrFail($alloc['invoice_id']);
            if ($alloc['amount'] > $invoice->remaining_amount) {
                throw new OverAllocationException(
                    "充当額が請求残高を超えています（EDGE-005）: 請求書#{$invoice->invoice_number}"
                );
            }
            PaymentAllocation::create([
                'payment_id'       => $payment->id,
                'invoice_id'       => $invoice->id,
                'allocated_amount' => $alloc['amount'],
            ]);
            $invoice->decrement('remaining_amount', $alloc['amount']);
            $payment->decrement('unallocated_amount', $alloc['amount']);
        }

        return $payment->fresh();
    });
}
```

### 4. PaymentService::cancel 🔵

**信頼性**: 🔵 *要件定義 EDGE-009 より*

```php
public function cancel(Payment $payment, User $actor): void {
    DB::transaction(function () use ($payment, $actor) {
        foreach ($payment->allocations as $alloc) {
            $alloc->invoice->increment('remaining_amount', $alloc->allocated_amount);
            $alloc->delete();
        }
        $payment->update([
            'unallocated_amount' => $payment->amount,
            'cancelled_by'       => $actor->id,
            'cancelled_at'       => now(),
        ]);
    });
}
```

## 単体テスト要件

### テストケース1: 入金登録と分割充当 🔵

**Given**: 残額 50000 の請求書A、残額 30000 の請求書B、入金額 70000
**When**: `PaymentService::register()` で A に 50000、B に 20000 を充当
**Then**: 請求書A残額=0、請求書B残額=10000、Payment.unallocated_amount=0

### テストケース2: 充当超過エラー（EDGE-005） 🔵

**Given**: 残額 10000 の請求書
**When**: 20000 を充当しようとする
**Then**: `OverAllocationException` がスローされ DB は変更されない

### テストケース3: 消込取消で残額が復元される（EDGE-009） 🔵

**Given**: 充当済みの入金（請求書A に 5000 充当）
**When**: `PaymentService::cancel()` を呼ぶ
**Then**: 請求書A の remaining_amount が 5000 増加し、PaymentAllocation が削除される
