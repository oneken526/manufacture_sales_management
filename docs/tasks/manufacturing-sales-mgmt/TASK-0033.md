## タスク概要

入金の登録・一覧・消込（分割充当）画面を実装する。入金登録画面では得意先・入金日・金額を登録し（REQ-042）、消込画面では未充当の入金に対して複数請求書へ分割充当できる（REQ-043、ヒアリングQ6）。消込取消も画面から実行可能にする。アクセスは admin / sales に限定。

## 実装詳細

### 1. PaymentController 🔵

**信頼性**: 🔵 *Webルート仕様より*

**実装ファイル**: `app/Http/Controllers/PaymentController.php`

```php
public function store(PaymentRequest $request) {
    $customer = Customer::findOrFail($request->customer_id);
    $payment  = $this->paymentService->register(
        $customer,
        $request->only('payment_date', 'amount', 'payment_method', 'notes'),
        $request->input('allocations', []),
        auth()->user()
    );
    return redirect()->route('payments.show', $payment)
        ->with('success', '入金を登録しました');
}

public function allocate(Payment $payment) {
    $invoices = Invoice::where('customer_id', $payment->customer_id)
        ->where('remaining_amount', '>', 0)
        ->orderBy('invoice_date')
        ->get();
    return view('payments.allocate', compact('payment', 'invoices'));
}
```

### 2. 消込フォーム（Blade） 🔵

**信頼性**: 🔵 *要件定義 REQ-043・ヒアリングQ6 より*

**実装ファイル**: `resources/views/payments/allocate.blade.php`

- 未充当残高（payment.unallocated_amount）を表示
- 得意先の未消込請求書一覧（請求番号・請求残額）を表示
- 各請求書に充当金額入力欄を設ける
- JavaScript で充当合計が未充当残高を超えないようリアルタイムチェック

### 3. ルート設定 🔵

**信頼性**: 🔵 *Webルート仕様より*

```php
Route::middleware(['auth', 'role:admin|sales'])->group(function () {
    Route::resource('payments', PaymentController::class)->only(['index', 'show', 'create', 'store']);
    Route::get('/payments/{payment}/allocate',  [PaymentController::class, 'allocate']);
    Route::post('/payments/{payment}/allocate', [PaymentController::class, 'postAllocate']);
    Route::post('/payments/{payment}/cancel',   [PaymentController::class, 'cancel']);
});
```
