## タスク概要

dompdf（barryvdh/laravel-dompdf）のラッパーサービス `PdfService` を実装する。見積書・納品書・請求書の各 PDF 出力で共通利用するファサードラッパーであり、フォント設定（日本語対応）・用紙設定・ファイル名生成を一元管理する。EDGE-003 対応のスナップショット（PDF 生成時点のデータ固定）仕様もここで定義する。

## 実装詳細

### 1. PdfService 🔵

**信頼性**: 🔵 *要件定義 REQ-053・設計文書より*

**実装ファイル**: `app/Services/PdfService.php`

```php
class PdfService {
    public function generate(string $view, array $data, string $filename): \Illuminate\Http\Response {
        $pdf = \Barryvdh\DomPDF\Facade\Pdf::loadView($view, $data);
        $pdf->setPaper('A4', 'portrait');
        return $pdf->download($filename . '.pdf');
    }

    public function stream(string $view, array $data): \Illuminate\Http\Response {
        $pdf = \Barryvdh\DomPDF\Facade\Pdf::loadView($view, $data);
        $pdf->setPaper('A4', 'portrait');
        return $pdf->stream();
    }
}
```

### 2. dompdf 設定 🔵

**信頼性**: 🔵 *要件定義 REQ-052 より*

**実装ファイル**: `config/dompdf.php`（公開 or 上書き設定）

```php
'options' => [
    'font_dir'             => storage_path('fonts/'),
    'font_cache'           => storage_path('fonts/'),
    'default_font'         => 'ipaexg',  // IPAex ゴシック
    'isRemoteEnabled'      => true,
    'isHtml5ParserEnabled' => true,
    'chroot'               => base_path(),
],
```

### 3. AppServiceProvider への登録 🔵

**信頼性**: 🔵 *Laravelサービスコンテナパターンより*

```php
// app/Providers/AppServiceProvider.php
$this->app->singleton(PdfService::class);
```

## 実装手順（DIRECT）

1. `php artisan vendor:publish --provider="Barryvdh\DomPDF\ServiceProvider"` で設定ファイル公開
2. `config/dompdf.php` を日本語対応設定に修正
3. 日本語フォントファイルを `storage/fonts/` に配置（IPAex ゴシックまたは Noto Sans）
4. `app/Services/PdfService.php` を作成
5. `AppServiceProvider` に `PdfService::class` を singleton 登録
